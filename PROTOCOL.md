# PROTOCOL — 클론 프로젝트에 clone-kb 시스템 주입하기 (에이전트용 규약)

> 어떤 클론 프로젝트든 이 문서 하나로 무기고 시스템에 접속한다. **정본은 이 파일 — 프로젝트에는 포인터만 복사.**

## 0. 이 시스템이 뭔가
- **무기고**: `techniques/`(기법 카드 30+, status: experimental→verified→standard→retired) · `pipelines/`(조립도) · `ledger/`(평가 원장) · `cases/`(캠페인별 진행+루프 도식) · `status/`(무인 런 라이브 상태판 — 덮어쓰기·휘발) · `runs/`(**세션별 런 매니페스트 — append-only 축적.** 로딩 기법+세션 로직 도식+로직 평가. 런끼리 diff=하네스 진화 기록)
- 원칙: 대시보드=파생물(직접 편집 금지, `scripts/gen_dashboard.py`) · 승격/은퇴=Issue 제안→오너 승인 · standard만 skills/ 보유(강등 시 스킬도 제거)

## 1. 주입 절차 (프로젝트당 1회)
1. 이 리포를 로컬 클론(없으면): `git clone https://github.com/csbakk/clone-kb.git ~/Documents/project/clone-kb`
2. 대상 프로젝트의 진입 문서(HANDOFF.md 또는 CLAUDE.md) 상단에 아래 블록 삽입:
   ```markdown
   ## 무기고 (세션 시작 시 먼저)
   **기법 레지스트리 = https://github.com/csbakk/clone-kb** (로컬 ~/Documents/project/clone-kb)
   — 캠페인 시작 전 `index.md`에서 관련 기법 카드만 로드(스킬 로딩 패턴). 운영 의식은 `PROTOCOL.md` §2.
   ```
3. `cases/<프로젝트>.md` 생성(스키마는 기존 케이스 참조): 외부 폴더 경로·현재 상태·루프 도식(mermaid)·잔여 큐
4. `status/<프로젝트>.md` 생성(status/canvas.md 형식 복사): 무인 런 라이브 상태판
5. clone-kb README 재생성(`python3 scripts/gen_dashboard.py`) 후 push (`gh auth switch --user csbakk` → push → `gh auth switch`)
6. **대상 프로젝트 repo의 `README.md` = 산출물 대시보드로 교체**(프레임워크 보일러플레이트 금지). 리포트·갤러리·핸드오프·상태판(clone-kb `status/`)·무기고(clone-kb) 링크 + `.html`은 "로컬에서 열기" 안내. §2.5 참조.

## 2. 운영 의식 (매 세션 — 오케스트레이터 의무)
| 시점 | 할 일 |
|---|---|
| **캠페인 시작** | clone-kb `index.md` → 이번 작업 관련 카드만 로드 + **`runs/<날짜>-<캠페인>-<주제>.md` 런 매니페스트 생성**(로딩 기법+선택 근거 표·세션 로직 mermaid 도식·안전경계) — "이번 세션은 이 로직으로 돈다"를 시작 시점에 고정 |
| **무인 런 중** | **마일스톤/아크 단위 + 런 시작·종료 시** `status/<프로젝트>.md` 갱신+push(이벤트마다 X — 커밋 노이즈 방지, 오너결정 2026-07-16). 헤더 🔴 가동/⏸ 중단/⚪ 대기, 페이즈 mermaid·가동 에이전트 표·티켓 보드·이벤트 타임라인 |
| **세션 결산** | ①`ledger/`에 사용 기법 판정 append(성과/실패/중립+증거 링크) ②**런 매니페스트 §4 로직 평가 채움**(작동한 것/병목/다음 런에서 바꿀 것 — 세션 '로직 자체'의 평가, ledger·승격의 근거) + status:running→done ③**아침 보고 HTML**(무인 런일 때): 프로젝트 repo `reports/<날짜>-morning.html` 자체완결 1장 — **고정 8섹션**: ①헤더(세션·커밋범위·MD 정본 링크) ②판정 카드 4(게이트·핵심 수치·발견·안전) ③수치 추이 ④시각 증거(트리아지류 자료가 있으면 발췌 임베드+원본 링크 — 별도 요청 불요) ⑤밤 사이 발견 표 ⑥오너 액션 큐(체크박스, 결정 필요한 것만) ⑦다음 밤 큐 ⑧용어 사전(**접기 details**) + 본문 반복 용어 점선 툴팁(CSS data-tip — 네이티브 title 금지, 지연·미발화) — 오너가 모르는 용어 없이 읽히는 게 기준. **결정 회수**: 판정형 선택지는 반드시 **라디오 버튼**(항목당 1개 강제 — 체크박스 금지, 오너 혼선 실사고 2026-07-14), 내보내기 텍스트엔 **항목 제목 라벨 필수**(제목 없는 "☑"만 나열 금지). 모든 선택·메모는 localStorage 퍼시스트 + 우하단 "📋 결정 복사" 버튼(클립보드 → 오너가 채팅에 붙여넣으면 반영, 메모는 기본값에서 바뀐 것만). 정적 HTML 선택은 자동 전달되지 않음을 오너에게 고지. **시트는 완전한 HTML 골격(head/body) 필수 + 결산 전 헤드리스 렌더로 툴팁·내보내기 실검증**(주입 무적용 실사고 2026-07-14). ⚠스크린샷 포함이므로 **private 프로젝트 repo에만**, 이 KB(public)엔 경로 포인터만 ④`cases/<프로젝트>.md` 진행·도식 갱신 ⑤`gen_dashboard.py` 재생성 ⑥push. **역할 분리 원칙: HTML=판독·비교 뷰(레이아웃 제어) / MD=기록·축적 정본(diff·검색·링크)** |
| **새 기법 탄생** | experimental 카드 등록 → 2프로젝트 실증 시 verified → **Issue로 standard 승격 제안**(오너 승인) |
| **기법이 배신** | 반례를 ledger에 기록, 반복되면 강등/은퇴 Issue (retired는 `superseded_by` 필수) |

## 2.5 크로스캠페인 운영 결정 (누적 — 모든 클론 세션 적용)
> 한 캠페인에서 확립됐지만 **전 캠페인 공통 규칙**. 새 세션은 여기부터 읽고 준수. append-only.

- **(2026-07-16) 상태 갱신 = 마일스톤/아크 + 런 시작·종료** (이벤트마다 X). §2 참조.
- **(2026-07-16) 각 클론 프로젝트 repo README = 산출물 대시보드** (프레임워크 보일러플레이트 교체 필수). 리포트·갤러리·핸드오프·상태판(clone-kb)·무기고를 링크. clone-kb README(무기고)와 **상호 크로스링크**. 진입점 하나로 다 도달. 주입 절차 §1에 포함.
- **(2026-07-16) 크레딧 정본 = MCP 거래내역(transactions/balance)만.** 빌더의 bridge-history·표시비용 추정 불신(canvas서 실제 17.5cr을 ~70cr로 오산해 헛경보 발생). 생성 전후 MCP로 실측.
- **(2026-07-16) 멀티탭 CDP 안전.** 한 dev 포트에 탭 여럿이면 URL-substring 매칭이 비결정적으로 엉뚱한 탭에 attach → 데이터 손상. 탭당 1빌더, canvas-id 등으로 고정. harness는 다중매칭 시 **시끄럽게 실패**하게(조용히 하나 고르기 금지).
- **(2026-07-16) `.html` 산출물(갤러리·보고) = 로컬뷰.** GitHub에선 raw로 보임 — README에 "로컬에서 열기" 안내. (웹 렌더 필요 시 public pages repo 분리는 케이스별 결정.)
- **(2026-07-19) 보고 시각정보는 그 페이지에 임베드 — 분리 금지.** 클론·검수·버그·파리티·**결정/설명** 어떤 보고든 실물↔클론 비교 시각 증거(나란히 캡처)나 다이어그램을 **별도 HTML로 분리하지 말고 보고/결정 페이지 자체 안에** 넣는다(오너 지시: 결정 페이지와 시각 설명을 두 파일로 나눴다가 지적받음). 스샷 없으면 CSS 목업/인라인 SVG로라도 시각화. 텍스트·수치만으로 "일치함/해결됨" 보고 금지. 캡처는 프로젝트 private repo의 `reports/assets/<주제>/`에(clone-kb는 public이라 스샷 금지, 규칙만).
- **(2026-07-19) 보고 이미지는 클릭 확대(라이트박스) 필수.** 썸네일은 폭 맞춰 축소하되 클릭 시 풀스크린 확대(`object-fit:contain`·캡션·✕/Esc 닫기·`cursor:zoom-in`). 작은 썸네일만 주면 세부 판독 불가 → 오너가 눈으로 확인 못 함. 재사용 스니펫 `#lb` 오버레이 + `figure img` 클릭 핸들러.
- **(2026-07-20) ★WebGL/캔버스 앱에선 CDP 합성입력이 아예 안 먹는다 — 그걸로 내린 "동작 결론"은 불완전이 아니라 *틀릴 수* 있다.** canvas 실물(Higgsfield)에서 `Input.dispatchMouseEvent` 계열 합성 입력은 캔버스에 **아무 반응도 만들지 못한다**(픽셀 해시 프로브로 확인: 360°×1° 회전 스윕·40스텝 드래그 전부 무변화). 문제는 여기서 멈추지 않고, **"합성입력으로 안 되더라 → 실물은 그 제스처를 안 쓰나 보다"는 잘못된 추론**으로 이어졌다는 것: 정찰이 "실물 선 그리기 = 클릭→이동→클릭(드래그 아님)"이라 결론냈으나, **OS-신뢰입력(Quartz `CGEventPost(kCGHIDEventTap)`)으로 재측정하니 정반대** — 실물도 드래그로 그려지고 클릭→클릭은 4회 모두 아무것도 안 그렸다. 클론은 이미 드래그 방식이라 **브리프대로 "고쳤다면" 오히려 회귀**였다. 같은 기법으로 그동안 "미확정"이던 스마트 가이드라인도 최초 재현·실측했다(색 `#E05634` 등). **규칙: 캔버스/WebGL 표적의 *동작* 주장은 OS-신뢰입력으로 재현한 것만 확정으로 취급하고, CDP 합성입력 결과는 "측정 실패"로 기록한다(부재의 증거로 쓰지 않는다).** 창 타겟팅: `CGWindowListCopyWindowInfo`로 창 bounds + `window.screenX/screenY` + `outerHeight-innerHeight`로 콘텐츠 오프셋. 물리 커서를 움직이므로 짧게·상태 복구.
- **(2026-07-20) 검증 게이트 = 프로젝트의 실제 빌드 명령. `tsc --noEmit`만으론 불충분.** canvas에서 워커·오케 모두 `npx tsc --noEmit` exit 0으로 "통과" 판정하고 4커밋을 push했으나, 실제 `npm run build`(= `tsc -b && vite build`)는 **깨져 있었다** — `tsc -b`는 프로젝트 참조 config(`tsconfig.app.json`)를 쓰고 `noUnusedLocals` 같은 더 엄격한 옵션이 켜져 있어 미사용 import를 에러로 잡는데, 루트 `tsconfig.json` 기준의 `--noEmit`은 안 잡았다. **규칙: 완료 선언 전 반드시 `npm run build`(또는 프로젝트의 실제 빌드 스크립트)를 돌린다.** 워커 브리프에도 "`npm run build` exit 0"을 게이트로 명시할 것 — `vite build`만 따로 돌리는 것도 불충분(그건 타입체크를 건너뜀). 교훈의 일반형: **게이트는 "내가 고른 검사"가 아니라 "실제 배포 경로가 돌리는 검사"여야 한다.**
- **(2026-07-19) 결정 UI엔 자유 입력란 필수(오너 필수 지침).** 라디오/선택지 마지막에 **"기타 — 내 의견 직접 입력" 자유텍스트 필드**를 반드시 둔다 — 오너가 AI 제안 중 아무것도 안 고르고 자기 의견을 직접 적을 수 있게(AskUserQuestion "Other"와 동형). `__OTHER__` 라디오+`<textarea>`, 타이핑 시 자동선택, 복사 텍스트엔 `[내 의견] …` 반영. 제안만 나열=오너 사고를 선택지에 가둠, 금지.

## 3. 안전
- 이 리포는 **PUBLIC** — 회사·타사 정보, 실물 앱 자산(SVG·스크린샷·덤프), 토큰 절대 금지. 전부 포인터/경로만.
- 프로젝트 실자산(립 덤프 등)은 각 프로젝트 repo(private)에.
