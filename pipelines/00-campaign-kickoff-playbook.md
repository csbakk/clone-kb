# 캠페인 킥오프 플레이북

> **새 클론 캠페인 첫 세션에서 AI가 이 문서를 로드하고, 오너에게 아래 순서로 가이드한다.** 3캠페인(akiflow·canvas·notion)의 시행착오를 응축한 정본. 세부는 각 [[기법 카드]]로.

## 0. 시작 전 3질문 (이걸 안 정하고 출발한 게 최대 비용이었다)

1. **북극성 = 판정 수단**: "완료를 무엇으로 검사하는가?"를 한 문장으로. *판정 수단이 아키텍처를 결정한다* — "스크린샷 구분 불가"면 스타일-먼저도 통하지만, "DevTools로 요소를 찍어도 구분 불가"면 구조-우선 외에 답이 없다. notion에서 이걸 늦게 정해 7이터레이션 지불.
2. **대상 진단**: 문서형(notion — DOM 골격이 본체) / 캔버스형(canvas — 노드그래프·생성 파이프라인이 본체) / 목록·일정형(akiflow). 유형이 1순위 기법을 정한다.
3. **결정 라우팅 합의**: 사실판단=AI가 측정으로 종결, 방향/아키텍처 갈림길(복사 순서·SoT·저장 구조)=반드시 오너 결정으로 승격. AI가 조용히 정하면 안 된다(notion 실책).

## 1. 표준 순서 — 전층 클론 지도 ([[techniques.structure-first-cloning]] 정본)

```
정찰(read-only) → 회귀망(게이트) 먼저 → ①DOM 골격(셸까지 1:1, B안) → ②스타일(computed 실측 포팅)
→ ③상태 매트릭스(hover/press/focus) → ④이벤트 리스너 지도(동작의 골격) → ⑤애니메이션·⑥타이밍 지문
→ ⑦API 계약 파리티 → ⑧AX트리 전수 → ⑨엣지 상태(빈/로딩/에러)
```

- **①이 무조건 1순위(오너 채택 도크트린)**: 골격이 같아야 스타일이 공짜로 따라오고 보정 스택이 안 생긴다. 골격 = **래핑 체인 전체**(잎→부모→…→body, 층마다 태그·role·display·position·"층의 담당"[보더/배경/CE경계가 어느 층인가]·층수 자체). 셸은 공통 조상이라 1회 추출.
- **하네스 먼저**: 코드보다 게이트([[techniques.regression-harness-suite]])·측정 도구를 먼저. 무인 런의 전제조건.
- 이미 스타일-먼저로 진행된 캠페인은 **retrofit**(블록 단위 골격 교체+픽셀 무하락 확인) — canvas에서 실증(-40% diff 동반 감소).

## 2. 무엇이 왜 좋았나 (기법 선택 가이드)

| 상황 | 쓰는 기법 | 왜 |
|---|---|---|
| 파리티 측정 기반 | [[techniques.dom-first-measurement]] + 동일조건 캡처(같은 브라우저·사이드바 접힘·크롭 대칭·**캡처 상태 assert**) | 픽셀 스샷 단독은 retina/조건 드리프트로 거짓말함 |
| 수치화·회귀감시 | [[techniques.pixel-diff-baseline]](배지+히트맵+겹쳐보기) | 오너 육안 지적 루프를 수치로 대체. 절대값 아닌 기준선으로만 해석 |
| 대상에 API가 있으면 | **API 계약 클론**(notion: 클론이 공개 API를 로컬에서 말하게) | 동일 요청→양쪽 비교 = UI 자동화 없이 결정적 파리티. 데이터 주입도 이 경로로 |
| 실물 UI 자동화 | ❌ 기본 금지 | 실물·클론 양쪽 다 취약(정크 생성·빈 페이지). API/캐시 덤프로 대체 |
| 갭 수복 | measure-then-fix + [[techniques.rip-repair-loop]] | 실측 근거 없는 px 변경 금지. 수렴이면 억지 수복 말고 지도만 |
| 검증 | [[techniques.adversarial-verification]] + 게이트 티어링(T0 매변경/T1 스팟/T2 경계) | 빌더 자가선언 금지. T2는 mutation-heavy 경계만(비싸다) |
| 영속 | 파일 SoT(1페이지=1폴더, 브라우저 프로필 독립) | localStorage 단독은 프로필에 갇힘 — "주소만 있으면 어느 브라우저든" 요건 |
| 워커 위임 전 | **verify-before-dispatch**: 30초 grep으로 현 구현 확인 | 문서는 상한다 — stale 문서 믿고 헛워커 3건 낼 뻔(notion) |

## 3. 과거 실수 카탈로그 (같은 돈 두 번 내지 마라)

1. **★최대 실수 — computed 복사(화면만 복사)로 출발**: 픽셀 겹침 96%까지 갔는데도 **오너가 DevTools로 골격을 열어보니 사실상 하나도 일치하지 않았다** — 제목(태그·role·패딩 축 전부 다름), 리스트(마커/텍스트 부모 분리), 거터(이중 예약), 테이블(보더 층), 래퍼 체인. 픽셀 수치는 "다른 골격들이 보정으로 상쇄한" 결과였고, 그 보정 스택 때문에 내용이 바뀌면 깨졌다(번호목록 높이 붕괴). **픽셀 지표가 높다 ≠ 복사가 됐다 — 골격 검사 없는 픽셀 지표는 거짓 안심이다.** 두더지잡기 7이터레이션 + 골격 전면 retrofit 비용을 냈다. 규칙: 구조-우선(§1)이 무조건 선행, 픽셀 배지는 구조 게이트와 반드시 쌍으로.
2. **AI가 갈림길을 조용히 결정**(복사 순서) → 킥오프 3질문(§0)으로 예방.
3. **실물 UI 자동화 시도** → 실물에 정크 카드, 클론에 빈 페이지. API로 피벗이 정답이었다.
4. **파리티 루프 무한 순환** → 실물에 중복 184개. 규칙: distinct 세트 1회 생성, 루프 금지. 전용 루트 페이지 하위에만 생성(통째 아카이브 가능).
5. **뷰어 링크를 원장 last-wins로** → 스크린샷과 다른 회차 링크. 규칙: 링크·수치는 **같은 폴더의 산출물에서** 직접.
6. **브라우저 프로필 함정**: localStorage 데이터는 프로필 소속 — 다른 브라우저에서 링크가 "빈 페이지". 파일 SoT로 해소.
7. **Chrome 자동업데이트 중첩**(프레임워크 버전 4개) → 새 인스턴스 즉사 연쇄. 진단법: Versions/ 디렉토리 확인. 대책: 클론 작업은 Playwright 번들 크로뮴(시스템 Chrome 독립).
8. **탭 churn**: 새창/새페이지 반복 생성·파기 → 크래시 기여 + **노션 탭 재기동 반복 = 봇차단 위험**(오너 경고). 규칙: 기존 탭 navigate 재사용, 실물 세션은 프로필 쿠키로 보존, 로그인 화면 보이면 시도 없이 중단.
9. **측정 드리프트 방치** → 수치 불신. 규칙: 캡처 전 상태 assert(사이드바·스크롤·이미지로드·dpr)로 시끄럽게 실패 + `--repeat`로 분산 확인 + dpr 출처기록.
10. **워커가 백그라운드 던지고 턴 종료** → 결과 유실. 브리프에 "이번 턴 완결·PYTHONUNBUFFERED 포그라운드" 명문화, 재발 시 resume으로 회수.
11. **stale 문서 신뢰** → "누락"이라던 블록 4종이 이미 구현돼 있었다. verify-before-dispatch.
12. **무인 히드라**: 갭 수복마다 새 갭 발견 → 종결 규칙: "이번이 마지막 이터레이션, 신규 발견은 티켓만" 선언 후 마감 보고.
13. **오케가 보고 HTML을 작업 중 자동 오픈** → 오너 화면 방해. 최종 보고 시 1회만.
14. **병렬 워커 파일 충돌** → 스코프 배타 선언(파일 영역·실물 탭 접근) + 자기 파일만 정밀 스테이징.

## 4. 운영 규율 (무인 런)

- **모델 라우팅**: 오케=고급 모델(계획·브리프·독립검증·push·보고만), 실작업=하위 모델 워커. 오케 인라인 중노동 금지.
- **라이브 상태판**: 제품 repo push와 clone-kb `status/<캠페인>.md` push를 **한 세트로**. 런 시작 🔴/종료 ⚪.
- **push 루틴**: `gh auth switch --user csbakk` → push → switch back. main 항상 그린(게이트 통과분만).
- **보고**: 아침 HTML(라디오=택1 권장안 pre-check·체크박스=복수, data-tip 툴팁, 결정복사 버튼, 납품 전 헤드리스 검증).
- **결산 의식**: ledger append + status ⚪ + 기법 카드 승격/신설. 좋은 방법은 반드시 이 KB에 들어와야 다음 캠페인이 로드한다.
